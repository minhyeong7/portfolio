프로젝트 인사이트

이 과정에서 협업의 중요성과 팀 내 커뮤니케이션이 프로젝트 완성도에 얼마나 큰 영향을 미치는지를 직접 체감할 수 있었습니다.기능 구현 측면에서는 사용자 기능뿐만 아니라 관리자 전용 기능까지 포함한 서비스 흐름을 설계·구현하며, 다양한 기술 스택과 프레임워크를 활용해 안정적인 웹 서비스 구조를 구축하는 경험을 할 수 있었습니다.

특히 전체 시스템의 구조를 이해하고 각 기능이 어떻게 유기적으로 연결되는지를 고민하며 개발한 점이 인상 깊었습니다.또한 프로젝트를 진행하며 초기 단계에서의 설계와 개발 규칙 정의의 중요성을 크게 느꼈습니다.

사전에 시스템 구조와 개발 기준을 더욱 명확히 정리했다면, 개발 과정에서 발생한 시행착오를 줄이고 코드의 유지보수성도 더욱 높일 수 있었을 것이라 생각합니다. 이번 경험을 통해 테스트, 예외 처리, 리팩토링의 중요성을 인식하게 되었고, 향후 프로젝트에서는 보다 체계적인 설계와 검증 과정을 기반으로 개발을 진행하고자 합니다.

프로젝트 인사이트

MyBoard는 Spring Boot의 기본 구조와 웹 애플리케이션 전반의 흐름을 깊이 이해하기 위해 진행한 개인 프로젝트입니다. 단순 기능 구현에 그치지 않고, 게시판 서비스에 필수적인 인증·인가, 데이터 처리, 배포까지 전 과정을 혼자 설계하고 구현하는 데 집중했습니다.

게시판 CRUD 기능을 구현하며 Controller–Service–Repository 계층 분리와 책임 설계의 중요성을 체감했고, Thymeleaf를 활용한 서버사이드 렌더링을 통해 백엔드 로직과 화면이 어떻게 유기적으로 연결되는지 이해할 수 있었습니다. 또한 세션 기반 로그인과 구글 소셜 로그인 연동을 직접 구현하면서, 인증 흐름과 보안 처리에 대한 실무적인 감각을 키웠습니다.

개발 환경에서는 H2 데이터베이스를 사용해 빠른 개발과 테스트 환경을 구축했고, Render를 통해 배포하면서 로컬 환경과 운영 환경의 차이, 환경 변수 관리의 필요성을 경험했습니다. 이 프로젝트를 통해 단순히 기능을 만드는 것을 넘어, 웹 서비스가 실제로 동작하고 배포되는 전체 사이클을 이해하는 개발자로 한 단계 성장할 수 있었습니다.

  • 검색 오버레이 UI 및 자동 포커스 기능 구현
  • 슬라이드 사이드바 및 오버레이 UI 인터랙션 구현

프로젝트 인사이트

MyTube는 React와 Tailwind CSS를 활용해 영상 플랫폼 UI의 핵심 사용자 경험을 직접 구현해본 개인 프로젝트입니다. 카테고리 필터링 기능에서는 데이터 상태 변화에 따라 썸네일이 즉시 반영되도록 동적 렌더링 로직을 설계했으며, 이를 통해 React의 상태 관리와 컴포넌트 재사용성에 대한 이해를 높일 수 있었습니다.또한 검색 오버레이 UI에 자동 포커스 처리를 적용해, 사용자가 검색 흐름에 자연스럽게 몰입할 수 있도록 UX를 개선했습니다.

사이드바는 슬라이드 방식으로 구현하고, 오버레이 UI와 함께 제어하여 레이아웃 전환 시 사용자 동선이 깨지지 않도록 인터랙션을 설계했습니다. 이 과정에서 UI 상태 충돌을 방지하기 위한 조건 분기와 이벤트 처리의 중요성을 체감했습니다.

설문 기능의 경우 LocalStorage를 활용해 새로고침 이후에도 상태가 유지되도록 구현하며, 클라이언트 상태 지속성 관리에 대한 경험을 쌓을 수 있었습니다. 이 프로젝트를 통해 컴포넌트 단위 설계, UI 상태 흐름 관리, 그리고 사용자 중심 인터랙션 구현의 중요성을 실전에서 학습할 수 있었습니다.
